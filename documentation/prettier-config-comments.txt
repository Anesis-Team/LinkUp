Prettier Configuration:

- `trailingComma`: Use trailing commas where valid in ES5.
- `tabWidth`: Use 2 spaces for indentation.
- `semi`: Do not add semicolons at the end of statements.
- `singleQuote`: Use single quotes for strings.
- `quoteProps`: Only add quotes around object literal property names when necessary.
- `jsxSingleQuote`: Use double quotes for JSX attributes.
- `arrowParens`: Always include parentheses around arrow function parameters.
- `bracketSpacing`: Print spaces between brackets in object literals.
- `jsxBracketSameLine`: Place the > of a multi-line JSX element at the end of the last line.
- `endOfLine`: Maintain consistent line endings.
- `htmlWhitespaceSensitivity`: Treat HTML whitespace as CSS.
- `insertPragma`: Do not insert @format pragma into files.
- `requirePragma`: Do not require a special first comment to format.
- `proseWrap`: Maintain line breaks in prose (e.g., comments).
- `useTabs`: Use spaces instead of tabs for indentation.
- `max_line_length`: Limit lines to 100 characters.
